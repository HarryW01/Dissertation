\chapter{Introduction}

In this project, a Python programming environment is created targeted at new programmers. The environment builds upon the idea of helping users learn by removing some of the complexity behind programming, in turn creating a less overwhelming experience. In particular, this project focuses on heavily restricting the expressiveness of Python, then gradually increasing this in line with the user's knowledge. This is done by creating an environment that restricts both language features and built-in functions.

\section{Definitions}

A \textit{restriction} consists of removing syntax from the language; code containing restrictions cannot be run, just as code containing a syntax error cannot. Restrictions come in two forms: 
\begin{itemize}
    \item \textit{Language features} or just \textit{features}: 
    Programming language constructs and data structures (e.g. loops, conditionals, functions, lists).
    \item \textit{Built-in functions} or just \textit{built-ins}: The set of Python built-in functions defined in the Python documentation\footnote{\href{https://docs.python.org/3/library/functions.html}{https://docs.python.org/3/library/functions.html}}.
\end{itemize}

\section{Motivation}

Novice programmers face a variety of problems when first learning; creating new tools to help them overcome these challenges has been a focus of the computing education field.

\section{Related Work}



 The focus of many tools is to create a more rigid structure of programming, in the hopes that a more constrained environment will leave the learner more focused and less overwhelmed.

Many approaches exist for this: for example, Scratch\footnote{\href{https://scratch.mit.edu/}{https://scratch.mit.edu/}} does this by abstracting programming constructs into blocks that learners can slot together to create a program. In this case, the added structure comes from the blocks, which write the majority of the syntax for the user. By constraining users to these set blocks, they no longer have to remember long and complicated syntax, and can also see all the statements they could write in one menu. 

Another avenue in research is the use of visual features, which have long been shown to aid learning. This approach has been explored for programming purposes: for example, visualising code execution is the main idea behind learning environments such as Python Tutor\footnote{\href{https://pythontutor.com/}{https://pythontutor.com/}} or Thonny\footnote{\href{https://thonny.org/}{https://thonny.org/}}. Both programs make use of visuals in different ways. Python Tutor lays out all the necessary information a learner should track, reducing the intermediate values they have to memorise; Thonny isolates function calls by creating separate windows to visualise each one. The efficacy of adding visual features to learning environments thus makes a strong case for their inclusion in the project.

With these two approaches considered, this project aims to create a Python learning environment that fills in some of the gaps in previous concepts identified above. The core concept attempts to remove complexity in the language by restricting the features a user can use to exactly the subset of features that they have been taught. Consequently, users don’t become overwhelmed by the wealth of language features available to them.



Restriction reduce the scope for learners to go wrong. While ‘rabbit holes’ important for continued interest and learning, in the beginning fundamentals need to be taught to prevent users becoming overwhelmed.
